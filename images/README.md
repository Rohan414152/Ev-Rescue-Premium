# Image Guidelines for EV Rescue Premium

## Image Folder Structure
Place all your app screenshots in the `images/` folder with the following naming convention:

## Required Screenshots

### 1. Dashboard Overview (`dashboard.png`)
- Main dashboard view showing key metrics
- Fleet overview
- Real-time status indicators
- Recommended size: 1200x800px

### 2. Emergency Request Flow (`emergency-request.png`)
- Emergency charging request interface
- User flow for requesting help
- Request form and confirmation
- Recommended size: 1200x800px

### 3. Real-time Tracking (`live-tracking.png`)
- Live map with service provider locations
- Active emergency requests
- Real-time updates interface
- Recommended size: 1200x800px

### 4. Fleet Management (`fleet-management.png`)
- Fleet management dashboard
- Vehicle status overview
- Management controls
- Recommended size: 1200x800px

### 5. Charging Station Finder (`charging-finder.png`)
- Interactive map with charging stations
- Station details and availability
- Search and filter functionality
- Recommended size: 1200x800px

### 6. Analytics Dashboard (`analytics.png`)
- Performance metrics
- Charts and graphs
- Business insights
- Recommended size: 1200x800px

### 7. Mobile Responsive Design (`mobile-view.png`)
- Mobile interface screenshots
- Can be a collage showing multiple mobile screens
- Recommended size: 800x1200px (mobile ratio)

## Image Quality Guidelines

- **Format**: PNG or JPG
- **Resolution**: High resolution (at least 1200px wide for desktop views)
- **Quality**: Clear, crisp screenshots
- **Consistency**: Use the same browser/device for consistent styling
- **Content**: Use realistic but anonymized data

## Taking Screenshots

### For Desktop Views:
1. Open your app in a browser
2. Set browser to full screen or consistent window size
3. Use browser developer tools to ensure consistent sizing
4. Take screenshots of each major feature

### For Mobile Views:
1. Use browser developer tools mobile emulation
2. Select a standard device (iPhone 12/13 or similar)
3. Take screenshots in portrait orientation
4. Consider showing multiple screens in one image

## Optional Additional Images

You can also add:
- `hero-banner.png` - Main promotional image
- `architecture-diagram.png` - System architecture
- `user-flow.png` - User journey diagrams
- `features-overview.png` - Feature comparison or overview

## Updating README

After adding your images, the README.md will automatically display them. Make sure:
- Image file names match exactly (case-sensitive)
- Images are in the `images/` folder
- File formats are web-compatible (PNG, JPG, GIF)

## GitHub Display

- Images will display automatically on your GitHub repository page
- Users can click to view full-size versions
- Consider file sizes (keep under 1MB each for faster loading)
